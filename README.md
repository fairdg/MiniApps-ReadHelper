# ReadHelper

Telegram Mini App + бот, которые дробят книгу/статью на небольшие порции и
присылают их по расписанию — читать понемногу, не теряя фокус.

## Структура

- **`api/`** — Vercel serverless-функции (роуты): тонкие обработчики, вся
  логика — в `server/`.
- **`server/`** — бэкенд: репозитории (БД), дробление текста (Gemini),
  Telegram, извлечение статей по ссылке.
- **`src/`** — фронтенд (Vue 3 + Vite), сама мини-аппа.
- **`scripts/`** — разовые операционные скрипты (миграция БД, регистрация
  вебхука/команд бота).
- **`public/`** — статика Vite.

## Разработка

```bash
npm install
npm run dev        # фронтенд локально (api/ бьёт в прод через прокси в vite.config.js)
npm run migrate    # прогнать server/db/schema.sql на базу из .env.local
npm run set-webhook    # зарегистрировать вебхук бота в Telegram
npm run set-commands   # зарегистрировать /start, /help в меню бота
```

Секреты — в `.env.local` (не в git). Список нужных переменных — в
`DEVLOG.local.md`.

## Деплой

Пуш в `master` на GitHub → автодеплой на Vercel. Рассылка порций идёт по
внешнему крону (cron-job.org), который дёргает `/api/cron/deliver` каждые
15 минут — встроенный Vercel Cron на бесплатном тарифе ограничен раз в сутки.
