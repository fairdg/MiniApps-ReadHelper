# server/

Бэкенд-логика, используется только обработчиками из `api/`. Ничего отсюда не
импортируется фронтендом (`src/`).

- `db.js` — клиент Neon (Postgres).
- `db/schema.sql` — схема БД, накатывается через `npm run migrate`.
- `repositories/` — вся работа с БД (users, books, chunks, deliveries,
  feedback) — по одному файлу на таблицу/сущность.
- `chunking.js` — дробление текста книги на порции (главы + ИИ + фолбэк).
- `gemini.js` — вызов Google Gemini для смыслового дробления.
- `articleExtractor.js` — вытаскивание текста статьи по URL (jsdom +
  Readability).
- `textClean.js` — нормализация переносов строк, вырезание картинок,
  проверка что файл вообще похож на текст.
- `telegram.js` — обёртка над Telegram Bot API (`sendMessage`).
- `delivery.js` — общая логика отправки следующей порции (использует и крон,
  и ручной "отправить сейчас" в режиме разработчика).
