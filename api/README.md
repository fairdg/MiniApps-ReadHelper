# api/

Vercel serverless-функции — файл = роут (стандартная файловая маршрутизация
Vercel, поэтому папка обязана оставаться в корне проекта).

Обработчики тонкие: разбор запроса → вызов `server/` → ответ. Бизнес-логика
сюда не пишется, только в `server/`.

- `books/add.js` — добавление книги (текст/файл/ссылка), запускает дробление.
- `books/list.js` — список книг пользователя с прогрессом.
- `books/[id].js` — удаление книги.
- `books/[id]/chunks.js` — чанки конкретной книги.
- `books/[id]/delivery.js` — смена частоты уведомлений / паузы.
- `books/[id]/deliver-now.js` — ручная отправка (только владелец, режим разработчика).
- `cron/deliver.js` — точка входа для внешнего крона, рассылает просроченные порции.
- `telegram/webhook.js` — приём апдейтов от Telegram (`/start`, `/help`).
- `feedback.js` — приём отзывов из мини-аппы.
